# ACP stdio probe (Task 5.1) - GO

Date: 2026-07-17. CLI: grok 0.2.102. macOS. Probe: newline-delimited JSON-RPC
2.0 against `grok agent stdio --leader-socket <private>` spawned in a
throwaway private HOME (auth.json copy only; dir scrubbed after).

## Observations (frames captured live)

- `initialize` (protocolVersion 1) -> result with agentCapabilities:
  `loadSession: true`, promptCapabilities (embeddedContext), mcpCapabilities
  (http/sse), and `_meta` extensions: `x.ai/hooks` with
  `blockingEvents: ["pre_tool_use"]`, `decisions: ["deny"]` - a PER-TOOL-CALL
  VETO channel; `authMethods` include `cached_token` from the private home's
  auth.json (no interactive auth needed).
- `session/new {cwd, mcpServers: []}` -> sessionId + model state
  (grok-4.5, 500k context, reasoningEfforts low/medium/high).
- `session/prompt` (text) -> streamed `session/update` notifications
  (agent_message_chunk et al., 28 frames) then result
  `stopReason: "end_turn"` with full usage metadata (tokens, modelCalls,
  requestId).
- Clean terminate; no stderr noise; private home teardown clean.

## Go decision and consequences for Task 5.3

- GO: hardened-home compatible; cached-token auth works; streaming and usage
  arrive natively; `loadSession: true` enables reattach across wrapper
  processes.
- The pre_tool_use deny hook is the strongest per-session confinement
  primitive available - the peer channel design SHOULD register it and deny
  tool calls whose paths escape the session worktree (defense in depth in
  front of the OS sandbox).
- Framing is plain ndjson JSON-RPC 2.0 (no LSP Content-Length headers).
